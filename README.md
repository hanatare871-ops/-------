# デイトレ・エントリー復習AIアプリ

5分足・1分足のチャート画像をアップロードすると、AIがあらかじめ定義したデイトレ手法
(ブレイクアウト順張り + 急落・急騰の逆張り)に沿って「その日どこでエントリーすべき
だったか」を事後判定・解説するツールです。

**重要**: これは復習・パターン認識訓練用のツールであり、リアルタイムの売買判断
ツールではありません。事後分析であるため、判定精度は将来の予測力を意味しません。
投資助言ではありません。

## 成果物

| ファイル | 役割 |
|---|---|
| `entry-review-app.html` | アプリ本体(単一HTMLファイル、フレームワーク不使用)。ローカルでブラウザから直接開ける。 |
| `anthropic-proxy.gs` | Google Apps Script製のAPIプロキシ(A案)。ブラウザとAnthropic APIの間を中継し、APIキーをブラウザに露出させない。 |
| `README.md` | 本ファイル。 |

## セットアップ手順

### 1. GASプロキシをデプロイする(A案・推奨)

1. https://script.google.com/ で新規プロジェクトを作成する。
2. `anthropic-proxy.gs` の中身をすべてコピーし、エディタに貼り付けて保存する。
3. Anthropic Console (https://platform.claude.com/) でAPIキーを発行する。
4. GASエディタ左メニューの「プロジェクトの設定」→「スクリプト プロパティ」で
   キー名 `ANTHROPIC_API_KEY`、値に発行したAPIキーを登録する。
   (コードに直接APIキーを書き込まないこと)
5. 右上「デプロイ」→「新しいデプロイ」→種類「ウェブアプリ」を選択。
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員**
6. 「デプロイ」をクリックし、表示されたWebアプリURL
   (`https://script.google.com/macros/s/xxxxx/exec`) をコピーする。
7. コードを更新した場合は、既存デプロイを「新バージョン」で再デプロイしないと
   変更が反映されない点に注意。

### 2. アプリ側にURLを設定する

1. `entry-review-app.html` をブラウザで直接開く(ダブルクリックでOK)。
2. 「⚙ 設定」セクションを開き、API呼び出し方式が
   「A案:GASプロキシ経由(推奨・既定)」になっていることを確認。
3. 「GASプロキシ Web App URL」欄に手順1でコピーしたURLを貼り付け、
   「設定を保存」を押す。

### 3. 動作確認

1. 「📈 チャート画像」セクションで5分足チャートのスクリーンショットを
   アップロードする(ドラッグ&ドロップ / クリックして選択 / 貼り付けのいずれか)。
2. 「📝 補助情報」に銘柄コードや前日高値などを入力する(任意だが入力推奨)。
3. 「エントリーポイントを判定」を押す。数十秒待つとエントリー候補・見送り局面・
   総評が表示される。
4. 結果は自動的に「判定履歴」に保存され、リロード後も一覧から再表示できる。

## B案: ブラウザから直接呼び出す場合

「⚙ 設定」で「B案:ブラウザから直接呼び出し」を選び、Anthropic APIキーを
入力すると、GASを経由せずブラウザから直接 Anthropic API を呼び出せます。

- APIキーは **sessionStorage** にのみ保持され、タブを閉じる/リロードすると
  消えます(永続保存はしません)。
- ブラウザから直接APIキーを送信するため、共有端末や公開環境での利用は
  推奨しません。個人の検証用途を想定しています。
- CORSまわりの最新仕様(必要ヘッダー等)は
  https://docs.claude.com/en/api/overview を参照してください。

## 手法ルールのカスタマイズ

判定に使う手法ルールは `entry-review-app.html` 内の `METHOD_RULES` 定数に
まとめられています。ルールを調整したい場合はこの定数を直接編集してください
(v1では設定画面からの編集には対応していません)。

## データの扱い

- 判定履歴は直近50件までブラウザの `localStorage` に保存されます。
- 「JSONでエクスポート」で履歴全体をファイルにダウンロードできます。
- チャート画像そのものは履歴として保存されません(判定結果のJSONのみ保存)。
